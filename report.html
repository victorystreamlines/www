دليل إعداد SSH والاتصال بـ VPS

خطوات واضحة وسهلة لإعداد الاتصال بخادمك على Hostinger

1
تسجيل الدخول إلى VPS عبر Terminal
LocalHost Terminal (جهازك)

لماذا هذه الخطوة؟

أول خطوة للاتصال بالـ VPS هي تسجيل الدخول باستخدام SSH من جهازك. هذا يسمح لك بالتحكم الكامل في الخادم عن بعد.

الأمر المطلوب
ssh root@YOUR_VPS_IP

ملاحظة: استبدل YOUR_VPS_IP بعنوان IP الخاص بخادمك (مثال: 185.123.45.67)

استجابة النجاح المتوقعة
The authenticity of host '185.123.45.67' can't be established.
ECDSA key fingerprint is SHA256:xxxxxxxxxxxxx.
Are you sure you want to continue connecting (yes/no)?

اكتب: yes ثم اضغط Enter

خطأ محتمل: Connection Refused
ssh: connect to host 185.123.45.67 port 22: Connection refused
ماذا تفعل؟

• تحقق من عنوان IP صحيح
• تأكد من تشغيل VPS في لوحة Hostinger
• تحقق من Firewall لا يحجب المنفذ 22

2
إدخال كلمة المرور
LocalHost Terminal

لماذا هذه الخطوة؟

بعد تأكيد الاتصال، سيطلب منك Terminal إدخال كلمة مرور root. هذه كلمة المرور موجودة في لوحة Hostinger.

ستظهر لك:

root@185.123.45.67's password: _

⚠️ ملاحظة هامة: عند كتابة كلمة المرور لن تظهر أي أحرف على الشاشة (هذا طبيعي للأمان)

تسجيل دخول ناجح
Welcome to Ubuntu 22.04 LTS
Last login: Sat Oct 18 02:00:00 2025
root@vps-hostname:~#

✅ رائع! أنت الآن داخل VPS Terminal

خطأ: كلمة مرور خاطئة
Permission denied, please try again.
ماذا تفعل؟

• اذهب إلى لوحة Hostinger
• افتح VPS Management
• اضغط على "Reset Root Password"
• استخدم كلمة المرور الجديدة

3
تثبيت PHP SSH2 Extension على LocalHost
LocalHost Terminal (Laragon/XAMPP)

لماذا هذه الخطوة؟

PHP يحتاج إلى extension خاص للاتصال بـ SSH. بدون هذا الـ extension، سيظهر خطأ في index.php

للتحقق إذا كان مثبت:

الأمر المطلوب
php -m | grep ssh2
SSH2 مثبت
ssh2

✅ ممتاز! الـ extension موجود، انتقل للخطوة التالية

SSH2 غير مثبت
(لا يظهر أي شيء)
خطوات التثبيت في Laragon:

1. افتح Laragon
2. Menu → PHP → Extensions
3. ابحث عن ssh2 وفعله
4. أعد تشغيل Apache

أو يدوياً:
• افتح php.ini
• أضف: extension=ssh2
• أعد تشغيل Apache

4
تحديث وترقية النظام على VPS
VPS Terminal

لماذا هذه الخطوة؟

قبل أي عمل على VPS، يجب تحديث قوائم الحزم والنظام لضمان الأمان وآخر الإصدارات.

الأمر المطلوب
apt update && apt upgrade -y
التحديث نجح
Reading package lists... Done
Building dependency tree... Done
All packages are up to date.

✅ رائع! النظام محدث

5
التحقق من SSH Server يعمل
VPS Terminal

لماذا هذه الخطوة؟

للتأكد أن SSH Server (sshd) يعمل بشكل صحيح ويستقبل الاتصالات.

الأمر المطلوب
systemctl status sshd
SSH يعمل
● ssh.service - OpenBSD Secure Shell server
   Loaded: loaded
   Active: active (running)

✅ ممتاز! SSH Server يعمل

SSH متوقف
Active: inactive (dead)
تشغيل SSH:
أمر التشغيل
systemctl start sshd
6
فحص Firewall والمنافذ
VPS Terminal

لماذا هذه الخطوة؟

Firewall قد يمنع الاتصالات الخارجية. يجب التأكد أن المنفذ 22 (SSH) مفتوح.

الأمر المطلوب
ufw status
Firewall غير مفعل
Status: inactive

✅ جيد! لا توجد قيود، انتقل للخطوة التالية

Firewall مفعل
Status: active
فتح المنفذ 22:
أمر السماح
ufw allow 22/tcp

ثم أعد تحميل:

إعادة التحميل
ufw reload
✓
الآن جرب الاتصال!

بعد إتمام جميع الخطوات السابقة، ارجع إلى index.php وجرب الاتصال!
